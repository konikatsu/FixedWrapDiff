# FixedWrapDiff 開発作業記録

作成日: 2026-06-18

この文書は、FixedWrapDiff を新規作成して GitHub 公開、Release 配布まで進めた作業の流れを残すための記録です。

## 目的

WinMerge では、固定長テキストを指定桁数で仮想的に折り返して比較する用途に合いにくい。

そのため、長い固定長レコードを指定桁数ごとに分割し、左右または上下に並べて差分を確認できる最小構成の WinForms ツールを作成した。

## 最初に決めた方針

- アプリ名、プロジェクト名は `FixedWrapDiff`
- C# WinForms で作成
- .NET 8 を優先
- まずは動く最小構成を優先
- 元ファイルは変更しない
- 表示部品は `RichTextBox`
- 固定長の比較に必要な情報を仮想行として持つ
- 将来、文字単位の差分強調を追加しやすい構造にする

## 内部データ構造

比較用の中心データとして、以下の record を作成した。

```csharp
public sealed record VirtualLine(
    int OriginalLineNumber,
    int WrapIndex,
    int StartColumn,
    int EndColumn,
    string Text);

public sealed record CompareLine(
    VirtualLine? Left,
    VirtualLine? Right,
    bool IsDifferent,
    bool ExistsOnlyLeft,
    bool ExistsOnlyRight);
```

元ファイルを直接比較するのではなく、いったん `VirtualLine` のリストに変換してから、左右の同じ位置の仮想行を比較する構成にした。

## Release 配布の方針変更

最初は、.NET Runtime を同梱する self-contained build を Release に置いた。

しかし、zip 展開後のファイル数が非常に多くなるため、配布方針を軽量版に変更した。

現在の方針:

- Release には軽量版のみを置く
- 利用者 PC に .NET 8 Desktop Runtime がない場合は、起動時に .NET のダウンロード案内が表示される
- README に .NET 8 Desktop Runtime のダウンロードリンクを記載する

軽量版の publish コマンド:

```powershell
dotnet publish -c Release
```

Release 用 zip:

```text
FixedWrapDiff-v0.1.0-win-x64.zip
```

- self-contained build は利用者に優しいが、WinForms では配布ファイル数が非常に多くなる。
- 小さなツールでは、軽量版 + .NET Runtime 案内のほうが見た目も扱いやすい。
- README の最初にダウンロードリンクと画面イメージがあると、初見の人に伝わりやすい。
